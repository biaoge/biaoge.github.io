Steps to test iOS app on device

Posted on 2018-05-25

Before test:
Becoming an Apple developer is free, but this won’t enable you to submit an app to the App Store — to do that you must pay the aforementioned US$99 fee.

Steps:

  1. 需要一个App ID (不是Apple account)
  2. 创建证书请求文件Certificate Signing Request (CSR文件)
  3. 根据CSR文件创建开发者证书(CER文件)(开发，测试用的Developer证书)
  4. 添加设备
  5. 根据Devices创建Provisioning Profiles配置文件 (PP文件)
  6. 设置Xcode 然后真机调试

### About Certificate App IDs & Profiles, iTunes Connect: ###

1. Certificate App IDs & Profiles
对于没有越狱的ios来说，只有被Apple approve并且通过app store安装的app才能在真机上运行.
Apple通过要求每一个app都有一个signed Apple Certificate. 从App Store来的app都会捆绑一个Certificate, 系统会在app运行前verify这个Certificate.

There are two types of profiles:
Development profiles. These are tied to specific devices, so the app can only run on those.
Distribution profiles. These are used to sign your app before you submit it to Apple for approval. Although they contain no device-specific information, you can’t use them to install apps on any device yourself because Apple still has to sign the app after the approval process.

2. iTunes Connect
iTunes Connect is the portal you’ll use to submit an app. This is where you’ll register a new app, enter the app’s description and screenshots, choose the price, and configure game center and in-app purchases.

This is also the portal you’ll use to agree to new contracts, set your financial data (so you can bank that profit) and check your sales.

创建App ID

打开苹果开发者官网, 点击右边的Account (account记得买开发者账号的权限, 一年99USD)
点击Certificates, Identifiers & Profiles

创建App ID，点击右上角的+号

设置App ID和App ID Suffix。App ID Suffix这里选Explicit App ID，表示明确的app id和Xcode项目中的Bundle ID对应. Wildcard App ID则是通用的app id 不需要和项目中的Bundle ID对应

下面的App Services如果不要求iPhone提供额外的服务, 默认的就行, 然后点continue

点击register

创建证书请求文件(CSR文件)

在LaunchPad中找到KeyChain Access


点击keyChain下的Certificate Assistant


保存到自己想保存的地方(我这里是桌面上的一个文件夹), 记得输入自己的邮箱



生成 Certificates(CER文件)

首先你要生成两种Certificates, 一个是development profiles, 一个是disrtibution profiles

找到苹果开发者网站下的Account的Certificates(确保左上角的下拉菜单选中iOS, tvOS, watchOS), 点击All, 点击右上角的 + 号, 然后选择iOS App Development






在电脑上选择你之前创建的csr文件

下载生成的CER文件

添加设备

可以再ITunes中找到设备的UUID

添加设备成功

根据Device创建Provisioning Profiles配置文件(PP文件)

在apple developer account页面中选择Provisioning profiles, 点击All，然后右上角的+号, 选择iOS App Development

选择你之前创建的App ID





给PP文件写一个名字, 好记就行, 没有特殊要求

下载pp文件

打开Xcode, 进行真机测试, 千万注意修改target下的project以及projectName的Bundle ID, 然后把signing相关的都设置成自动









然后就可以在Xcode上进行实机测试了
